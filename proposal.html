제안 배경 및 목적

본 제안은 KCOC가 운영하는 서비스에 YouTube 동영상 시청 기반 포인트/토큰 보상 시스템을 도입하기 위한 개발 제안서입니다. (주)메스브레인은 이미 검증된 블록체인 지갑 솔루션을 보유하고 있으며, 이를 활용하여 개발 기간 단축 및 비용 절감을 실현합니다. 핵심 과제는 사용자가 영상을 실제로 시청하지 않고 보상을 수령하는 어뷰징(Abusing)을 기술적으로 차단하는 것입니다. YouTube IFrame Player API, 하트비트 시스템, 서버 사이드 검증을 연계하여 5단계 어뷰징 방지 아키텍처를 제공합니다.

블록체인 지갑 (토큰 보관·전송·내역)
YouTube 연동 + 포인트 보상 + 어뷰징 방지
상급 개발자 리딩 · 풀스택 구조
Section 02

기능 명세

7대 핵심 기능 모듈로 구성된 통합 플랫폼입니다.

1

🎬 YouTube 연동 동영상 재생

YouTube IFrame Player API를 활용한 영상 재생 기능. 커스텀 플레이어 UI를 제공하며, 영상 목록 관리 및 카테고리 분류를 지원합니다. 재생 상태(PLAYING/PAUSED/ENDED)를 실시간으로 추적합니다.

CORE
2

🪙 시청 기반 포인트 적립

영상을 시청할수록 포인트가 자동 적립되는 리워드 시스템. 영상별 지급 포인트 설정, 최소 시청률 임계값 설정, 중복 수령 방지 로직이 포함됩니다. 포인트 → 토큰 전환 기능을 제공합니다.

NEW
3

🚫 어뷰징 방지 시스템

5단계 다층 어뷰징 방지 아키텍처. 하트비트 신호 검증, 건너뛰기 감지, 배속 감지, 탭 비활성화 감지, 서버 사이드 최종 검증을 통해 허위 시청 데이터를 원천 차단합니다.

SECURITY
4

💼 토큰 지갑 시스템

기존 보유 지갑 솔루션 기반. 토큰 보관, 토큰 전송, 실시간 잔액 조회, 트랜잭션 내역, 통계 및 대시보드 기능을 포함합니다. 입출금 내역 CSV 다운로드를 지원합니다.

CORE (기존 활용)
5

⚙️ 어드민 관리 시스템

통합 관리자 패널. 유저 관리(조회/수정/삭제), 계정 잠금(Lock), 유저별 토큰·포인트 조정, 영상 관리(등록/수정/삭제), 보상 정책 설정을 지원합니다.

NEW

Google Authenticator 기반 TOTP 2단계 인증. 로그인, 출금, 민감 정보 변경 시 OTP 인증을 강제합니다. QR코드 설정 및 백업 코드를 지원하는 최고 수준의 보안을 제공합니다.

SECURITY

AWS/GCP 기반 비용 최적화 서버 구성. 오토스케일링, CDN 캐싱, DB 쿼리 최적화를 통해 최소 비용으로 최대 성능을 달성합니다. 24/7 모니터링 대시보드를 제공합니다.

Section 03

어뷰징 방지 아키텍처

5단계 다층 검증을 통해 허위 시청 및 포인트 부정 수령을 원천 차단합니다.

⚠️

클라이언트 단의 단순 이벤트 신호만으로 보상을 지급하는 방식은 패킷 위조, 자동화 봇 등으로 쉽게 우회될 수 있습니다. 본 제안은 서버 사이드 최종 검증을 핵심으로 하는 다층 방어 체계를 채택합니다.

🎯

레이어 1 · YouTube IFrame Player API 실시간 상태 추적

유튜브 공식 IFrame Player API를 통해 플레이어 상태를 실시간으로 모니터링합니다. onStateChange 이벤트로 PLAYING/PAUSED/ENDED 상태를 추적하고, getCurrentTime()과 getDuration()을 비교하여 실제 시청 비율을 계산합니다.

onStateChange getCurrentTime() getDuration() 실시간 추적
💓

레이어 2 · 하트비트(Heartbeat) 시스템

클라이언트는 5~10초 간격으로 서버에 "현재 N초 시청 중" 신호를 전송합니다. 서버는 이전 신호와 현재 신호 간의 시간 간격 vs 시청 진행량을 비교하여 물리적 시간 이상을 감지합니다. 예: 10초 경과 신호인데 시청 기록이 50초 증가 → 무효 처리

5~10초 주기 보고 서버 시간차 검증 이상 감지 시 무효화
⏭️

레이어 3 · 건너뛰기(Seeking) 및 배속 감지

사용자가 타임라인을 앞으로 건너뛰는 행위를 감지합니다. seek 이벤트를 모니터링하여 허용 범위를 초과하는 점프 시 해당 구간 시청을 무효 처리합니다. 플레이어 컨트롤러 숨김(controls=0)으로 UI 단에서 건너뛰기를 차단하며, getPlaybackRate()로 배속 재생 시 실제 시청 시간을 배속으로 나누어 적립합니다.

seeked 이벤트 모니터링 controls=0 차단 getPlaybackRate() 검증 배속 보정 포인트 계산
👁️

레이어 4 · Page Visibility API (활성 탭 감지)

사용자가 영상을 켜두고 다른 탭을 보거나 브라우저를 최소화하는 상황을 감지합니다. visibilitychange 이벤트와 document.hidden을 사용하여 페이지 비활성 시 영상을 자동 일시정지하고 시청 시간 측정을 중단합니다.

visibilitychange 이벤트 document.hidden 체크 비활성 시 자동 정지 시청 시간 측정 중단
🔒

레이어 5 · 서버 사이드 최종 검증 (Backend Validation)

모든 프론트엔드 데이터는 조작 가능하다는 전제하에 서버에서 최종 검증을 수행합니다. 시청 시작 시 고유 Session Token을 발급하고, 하트비트마다 갱신합니다. 영상 총 길이(L) 대비 최소 시청 비율(기본값: 90% 이상)을 충족해야만 포인트를 최종 지급합니다. 중복 요청 방지를 위한 Redis 기반 idempotency 키를 사용합니다.

JWT Session Token 검증 90% 이상 시청률 확인 Redis 중복 방지 비정상 패턴 자동 차단

Section 04

보안 아키텍처

Google OTP 기반 2차 인증을 포함한 다층 보안 체계를 적용합니다.

🔐
Google OTP (TOTP) 2차 인증

RFC 6238 표준 TOTP 기반 2단계 인증. 로그인·출금·어드민 접근 시 의무 적용. Google Authenticator 앱과 완전 호환.

🛡️
JWT + Refresh Token 체계

단기 Access Token (15분) + 장기 Refresh Token (7일) 이중 구조. 토큰 탈취 시 피해 최소화. Redis 블랙리스트 관리.

🔒
API Rate Limiting

IP·계정별 요청 횟수 제한으로 자동화 봇 차단. 하트비트 신호 과다 전송 시 차단. DDos 기본 방어 체계 포함.

🗄️
데이터 암호화

전송 구간 TLS 1.3, DB 민감 정보 AES-256 암호화. 개인정보 최소 수집 원칙. 비밀번호 bcrypt 해시 처리.

📋
감사 로그 (Audit Log)

모든 로그인·출금·어드민 작업에 대한 완전한 감사 로그 기록. IP·시각·행위자·변경 내용을 영구 보존.

🏗️
어드민 권한 분리

슈퍼어드민 / 운영자 / 뷰어 역할 구분. 최소 권한 원칙(Principle of Least Privilege) 적용. 계정별 2FA 의무화.
